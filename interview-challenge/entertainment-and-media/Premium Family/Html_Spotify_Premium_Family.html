Question 1:¶

"Spotify recently launched an update to the Family Mix playlist feature. You're given a dataset named FamilyData. Conduct an exploratory analysis on the family_mix_usage column to understand the impact of the recent update. What insights can you derive from the data regarding the usage of the Family Mix playlist after the update?"

Problem Solving Ideas and Tips:

  • Start by visualizing the distribution of the family_mix_usage to understand the average monthly plays.
  • Calculate summary statistics to get a sense of central tendency and spread.
  • A before-and-after comparison would be ideal, but since we have data from after the update, we can at least infer if many families are making good use of the Family Mix feature.
  • Outliers in the data may indicate families that are particularly engaged or disengaged with the feature.

Question 2:¶

Spotify is curious about the relationship between the length of time families have been subscribed (active_months) and their likelihood to churn (churned_last_month). Using the FamilyData dataset, create a model to predict the likelihood of churn based on the number of active months. Clearly state which feature is the target variable and its type.

Problem Solving Ideas and Tips:

  • The target variable is churned_last_month, which is binary. Therefore, we're looking at a binary classification problem.
  • A Logistic Regression is a good starting point for this kind of problem because of its simplicity and interpretability.
  • We should visualize the data to understand the distribution of churn across different months of activity.
  • It's essential to split the dataset into training and testing sets to evaluate the model's performance.

Question 3:¶

Your team is concerned about the revenue from families based in different regions. From the FamilyData dataset, write an SQL query to find the average monthly_payment for families from each family_region. However, there seems to be an error in the dataset: some families have a monthly payment of 0, which seems erroneous. Adjust your query to exclude these families. Additionally, upon visualizing the average payments in a bar graph, what insights or recommendations would you provide to the business?

Problem Solving Ideas and Tips:

  1. Data Filtration: First and foremost, ensure that the erroneous data (families with a monthly payment of 0) is excluded from our calculations.
  2. Aggregation: The key here is to aggregate the data by 'family_region' and compute the average 'monthly_payment'.
  3. Visualization: Visual representations can offer better insights than mere numbers. Plotting a bar graph can clearly show which regions are bringing in more revenue on average.
  4. Insights and Recommendations: Always relate insights back to the business problem. Recommendations can be based on focusing on high-revenue regions or improving marketing efforts in low-revenue regions.

Solution:

In [ ]:
import sqlite3
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

# Convert pandas dataframe to SQL table (Assuming df is the dataframe for FamilyData)
con = sqlite3.connect(":memory:")
df.to_sql("FamilyData", con, index=False)

# SQL Query to get average monthly_payment by family_region excluding families with monthly_payment = 0
query = """
SELECT family_region, AVG(monthly_payment) as avg_monthly_payment
FROM FamilyData
WHERE monthly_payment > 0
GROUP BY family_region
"""

avg_payments = pd.read_sql(query, con)

# Displaying the SQL results
print(avg_payments)

# Visualization using seaborn
plt.figure(figsize=(12, 6))
sns.barplot(data=avg_payments, x='family_region', y='avg_monthly_payment')
plt.title("Average Monthly Payment by Family Region")
plt.ylabel("Average Monthly Payment")
plt.xlabel("Family Region")
plt.show()

# Close the connection
con.close()
