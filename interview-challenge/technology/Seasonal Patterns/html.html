Business Problem Statement:¶

Booking is trying to leverage its vast troves of data to understand the seasonal booking patterns of its customers. By predicting the seasonal demand for hotel bookings across various regions, Booking can optimize its inventory, adjust marketing strategies, and provide better recommendations to its users. The challenge here is to uncover these patterns and design strategies based on time series analysis.

Step 2: Bootstrap:¶

Bootstrap involves drawing random samples with replacement from the data and then computing the statistic (mean, in this case) for each sample.

In [ ]:
import numpy as np

def bootstrap_sample(data, n_bootstrap):
    """Generate bootstrap samples from the data."""
    return np.random.choice(data, size=len(data), replace=True)

n_iterations = 10000
bootstrap_means = []

for _ in range(n_iterations):
    sample = bootstrap_sample(december_data['booking_id'], n_iterations)
    sample_mean = sample.mean()
    bootstrap_means.append(sample_mean)

Step 3: Calculate 95% Confidence Interval:¶

In [ ]:
confidence_level = 0.95
lower_percentile = (1 - confidence_level) / 2 * 100
upper_percentile = (1 + confidence_level) / 2 * 100

confidence_interval = (np.percentile(bootstrap_means, lower_percentile),
                       np.percentile(bootstrap_means, upper_percentile))

print(f"95% Confidence Interval for Mean Bookings in December: {confidence_interval}")

Step 2: Feature Engineering:¶

Temporal and lag-based features can help the model learn patterns and seasonality from the data.

In [ ]:
import numpy as np

# Creating lags
for i in range(1, 4):  # Creating 3 lags
    df_europe_daily[f'lag_{i}'] = df_europe_daily['number_of_bookings'].shift(i)

# Creating rolling features
df_europe_daily['rolling_mean_7'] = df_europe_daily['number_of_bookings'].rolling(window=7).mean()
df_europe_daily['rolling_std_7'] = df_europe_daily['number_of_bookings'].rolling(window=7).std()

# Extracting date features
df_europe_daily['day_of_week'] = df_europe_daily['stay_date'].dt.dayofweek
df_europe_daily['month'] = df_europe_daily['stay_date'].dt.month
df_europe_daily['quarter'] = df_europe_daily['stay_date'].dt.quarter

# Representing day_of_week and month as cyclical features
df_europe_daily['week_sin'] = np.sin(2 * np.pi * df_europe_daily['day_of_week'] / 7)
df_europe_daily['week_cos'] = np.cos(2 * np.pi * df_europe_daily['day_of_week'] / 7)
df_europe_daily['month_sin'] = np.sin(2 * np.pi * df_europe_daily['month'] / 12)
df_europe_daily['month_cos'] = np.cos(2 * np.pi * df_europe_daily['month'] / 12)


# Remove NaN values generated due to rolling features and lags
df_europe_daily.dropna(inplace=True)

# Log transformation
df_europe_daily['log_bookings'] = np.log1p(df_europe_daily['number_of_bookings'])
df_europe_daily.dropna(inplace=True)

# Splitting the dataset
train = df_europe_daily.iloc[:-30]
test = df_europe_daily.iloc[-30:]

Reasons of feature selections:

  • booking_date:

    • Gives temporal information. By extracting day, month, and day of the week, we can capture seasonality and trends in bookings.
    • Since day_of_week is a cyclical feature (i.e., it goes back to 0 after 6), it can be represented as two components: sin and cos, which can better capture its cyclic nature. This applies to month as well.

  • user_intent: It's important to understand the purpose behind bookings. Certain intents might show patterns, e.g., more 'Leisure' bookings on weekends.

  • Lagged features (like bookings from previous days): The number of bookings from the past few days can help predict future bookings, capturing autocorrelation in the data.

  • Rolling mean and standard deviation of the past week's bookings: This captures the trend and volatility in the recent past.

Step 4: More about Model Validation & Overfitting Prevention：¶

Time Series Cross-Validation: Instead of a simple train-test split, we can use TimeSeriesSplit from scikit-learn for cross-validation. This will give us a better idea of the model's robustness.

Regularization: For models that support regularization (like LASSO for linear regression or dropout for LSTMs), this can be applied.

Model Complexity: Given the vast number of features, tree-based models might provide better insights and interpretability. You can visualize feature importance to see which features have the most predictive power.

Comparison with a Naive Model: Compare the model's performance with a naive or simpler model. If the complex model isn't significantly better, it might be overfitting.

Residual Analysis: Analyze the residuals. They should ideally be white noise. Patterns in residuals indicate that the model has missed underlying patterns.

Final thoughts: The outlined approach should give us a robust framework for predicting bookings. Remember, model parameters (like (5,1,0) for SARIMAX) should be determined based on diagnostics and performance, not arbitrary choices. Consider using tools like pmdarima's auto_arima function to find optimal parameters.
