Question 1¶

Scenario: Microsoft Teams is exploring user engagement patterns, specifically those users who predominantly use text versus those who favor file sharing within chat conversations.

  • Question: Based on ChatActivity and SharedFiles, create a method to segment users into distinct behavioral clusters. Furthermore, if you were to visualize these clusters in a reduced dimension space, which techniques would you suggest?

# Creating features that represent user behaviors
user_behaviors = ChatActivity.groupby('userID').agg(
    textCount=('messageType', lambda x: sum(x == 'text')),
    fileCount=('messageType', lambda x: sum(x == 'file')),
    avgTextLength=('messageLength', 'mean')
).reset_index()

# To segment users into clusters
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler

# Scale the data
scaler = StandardScaler()
scaled_data = scaler.fit_transform(user_behaviors[['textCount', 'fileCount', 'avgTextLength']])

# Use KMeans to cluster
kmeans = KMeans(n_clusters=3) # Assumption: 3 clusters
user_behaviors['cluster'] = kmeans.fit_predict(scaled_data)

# For visualization in a reduced dimension space
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

pca = PCA(n_components=2)
reduced_data = pca.fit_transform(scaled_data)

plt.scatter(reduced_data[:, 0], reduced_data[:, 1], c=user_behaviors['cluster'])
plt.title('Visualization of Clusters in Reduced Dimension')
plt.show()

Question 2¶

Scenario: The effectiveness of threaded conversations is under the lens. There's a notion that certain threads initiate vibrant discussions, while others barely receive any responses.

  • Question: Using the ThreadedDiscussions and ChatActivity table, construct a model to forecast the likelihood of a thread receiving a response within 1 hour. Following your predictions, if there are instances where the model incorrectly forecasts, could you distinguish between instances where it falsely predicted a timely response versus when it inaccurately predicted a delayed response?

import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import confusion_matrix


# Step 1: Feature Engineering
# Here we assume that 'timestamp' in ChatActivity is the time the message was sent
# We will join the two tables on the 'chatID' column to get additional features
threaded_df = ThreadedDiscussions
chat_activity_df = ChatActivity

merged_data = pd.merge(threaded_df, chat_activity_df, on='chatID')

# Calculate the time difference between the thread initiation and each chat message in the thread
merged_data['timestamp'] = pd.to_datetime(merged_data['timestamp'])
merged_data['time_diff'] = (merged_data['timestamp'] - merged_data['timestamp'].min()).dt.total_seconds() / 3600

# Generate features such as the average message length, total number of messages before a response, etc.
features = merged_data.groupby('threadID').agg(
    avg_message_length=('messageLength', 'mean'),
    num_messages=('messageLength', 'count'),
    num_users=('userID', 'nunique'),
).reset_index()

# Step 2: Target Variable Creation
# Create a binary target variable indicating whether a thread received a response within 1 hour
threaded_df['timely_response'] = (threaded_df['responseTime'] <= 1).astype(int)

# Merge features and target variable
final_data = pd.merge(features, threaded_df[['threadID', 'timely_response']], on='threadID')

# Step 3: Model Construction
# Split data into training and test sets
X_train, X_test, y_train, y_test = train_test_split(
    final_data.drop(['threadID', 'timely_response'], axis=1),
    final_data['timely_response'],
    test_size=0.3,
    random_state=42
)

# Initialize and train a RandomForestClassifier
clf = RandomForestClassifier(random_state=42)
clf.fit(X_train, y_train)

# Predict the likelihood of a timely response on the test set
y_pred = clf.predict(X_test)

# Step 4: Error Analysis
# Confusion Matrix [TN, FP]
#                   [FN, TP]
cm = confusion_matrix(y_test, y_pred)

# False Positives: The instances where the model incorrectly forecasted a timely response
false_positives = cm[0][1]

# False Negatives: The instances where the model inaccurately predicted a delayed response
false_negatives = cm[1][0]

# Print Results
print(cm)
print('Number of False Positives:', false_positives)
print('Number of False Negatives:', false_negatives)
