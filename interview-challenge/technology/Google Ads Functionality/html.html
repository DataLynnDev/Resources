Using the WebContent and AdInformation tables, create a model to determine how well the content of a webpage matches with the keywords of ads displayed on them. What factors play a significant role in ensuring a good match?

Solution¶

Hints:

To determine how well the content of a webpage matches with the keywords of ads displayed on them, we need to follow these steps:

  1. Data Preprocessing:

    • Join the relevant tables to combine the content of the webpage with the keywords of ads displayed on them.
    • Tokenize and clean the content of each webpage and the keyword lists of ads. This involves removing stop words, lowercasing, stemming or lemmatizing, etc.

  2. Feature Engineering:

    • Calculate a match score for each webpage-ad pair. This could be done by counting the number of keyword occurrences in the content.
    • Use TF-IDF (Term Frequency-Inverse Document Frequency) or similar techniques to give more importance to rarer and potentially more relevant words.

  3. Modeling:

    • Given this is not a classic supervised learning problem with labeled data, unsupervised techniques or heuristic methods might be more apt. A simple model would be to score matches based on the TF-IDF-weighted overlap between ad keywords and webpage content.
    • To get an understanding of factors affecting a good match, one could try supervised models if we have some labeled data or use feature importance from tree-based models.

  4. Evaluation:

    • Visualize the top pages for a given ad based on match scores.
    • Comment on trends or patterns.

import pandas as pd
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import linear_kernel

# 1. Data Preprocessing:

# Join the tables on relevant columns
combined_df = df_user_interactions.merge(df_ad_information, left_on='AdClicked', right_on='AdID', how='inner')
combined_df = combined_df.merge(df_web_content, on='WebPageID', how='inner')

# 2. Feature Engineering:

# Convert KeywordList from list to string
combined_df['KeywordList'] = combined_df['KeywordList'].apply(lambda x: ' '.join(x) if isinstance(x, list) else x)

# 3. TF-IDF Vectorization:

# Initialize a vectorizer
vectorizer = TfidfVectorizer(stop_words='english', lowercase=True)

# Separate the content for ads and webpages
ads_content = combined_df['KeywordList']
webpages_content = combined_df['Content']

# Tokenize and transform each content using TF-IDF Vectorization
ads_tfidf = vectorizer.fit_transform(ads_content)
webpages_tfidf = vectorizer.transform(webpages_content)

# 4. Calculate the cosine similarities:

cosine_similarities = linear_kernel(ads_tfidf, webpages_tfidf)

# 5. Index mapping:
ad_indices = pd.Series(combined_df.index, index=combined_df['AdID'])

def get_top_matches(adID, cosine_similarities=cosine_similarities):
    idx = ad_indices[adID]
    sim_scores = list(enumerate(cosine_similarities[idx]))
    sim_scores = sorted(sim_scores, key=lambda x: x[1], reverse=True)
    sim_scores = sim_scores[1:11]
    page_indices = [i[0] for i in sim_scores]

    top_matches_df = combined_df.iloc[page_indices][['WebPageID', 'KeywordList', 'Content']]
    return top_matches_df

# Example: Top matches for AdID 4, change the number to view the connection of other AdIDs

From the UserInteractions table, calculate the probability that if a user spends more than a certain threshold of time (SessionTime) on a webpage, they are more likely to click on an ad.

Solution¶

Hints:¶

  1. Filter Data:

    • From the UserInteractions table, filter out rows where the SessionTime exceeds the specified threshold.

  2. Calculate Probabilities:

    • Calculate the probability that a user clicks an ad given they've spent more than the threshold time on a webpage.

  3. Visualization:

    • To provide a more comprehensive understanding, it may be useful to plot a graph that shows how the probability changes as the threshold increases.

import pandas as pd
import matplotlib.pyplot as plt

# Assuming the data is loaded in the respective dataframes
# df_user_interactions

# Setting an arbitrary threshold for demonstration
threshold = 10  # This can be any value depending on the dataset's range

# 1. Filter Data
filtered_data = df_user_interactions[df_user_interactions['SessionTime'] > threshold]

# 2. Calculate Probabilities
total_users_above_threshold = len(filtered_data)
users_clicked_ad_above_threshold = len(filtered_data[filtered_data['AdClicked'].notnull()])

probability = users_clicked_ad_above_threshold / total_users_above_threshold

print(f"Probability that a user clicks on an ad given they've spent more than {threshold} units of time on a webpage: {probability:.2f}")

# 3. Visualization
thresholds = range(1, int(df_user_interactions['SessionTime'].max()) + 1)
probabilities = []

for t in thresholds:
    filtered_data_t = df_user_interactions[df_user_interactions['SessionTime'] > t]
    total_users_t = len(filtered_data_t)
    users_clicked_ad_t = len(filtered_data_t[filtered_data_t['AdClicked'].notnull()])

    if total_users_t == 0:
        probabilities.append(0)
    else:
        probabilities.append(users_clicked_ad_t / total_users_t)

plt.plot(thresholds, probabilities)
plt.xlabel('Session Time Threshold')
plt.ylabel('Probability of Ad Click')
plt.title('Probability of Ad Click vs. Session Time Threshold')
plt.grid(True)
plt.show()

The script first calculates the probability of users who spent more than the set threshold time and clicked on ads. Next, to provide more clarity, it calculates these probabilities for a range of thresholds and visualizes them. The graph then assists in understanding how the likelihood of ad clicks changes as users spend more time on a page.

import pandas as pd
import matplotlib.pyplot as plt

# Assuming the data is loaded in the respective dataframes
# df_ad_information, df_user_interactions

# 1. Define Similar Campaigns:
# Let's say 'similar campaigns' are those that have certain keywords in common (for simplicity)
target_keywords = ["electronics", "books"]  # these should represent the keywords of the new campaign

# 2. Join Tables:
joined_data = df_user_interactions.merge(df_ad_information, left_on='AdClicked', right_on='AdID', how='inner')

# 3. Filter & Aggregate Data:
similar_campaign_data = joined_data[joined_data['KeywordList'].apply(lambda x: any(keyword in x for keyword in target_keywords))]

# Metrics:
# Click-through rate (CTR) = (Number of Ad Clicks / Number of Total Users who viewed the ad)
total_users_viewed = len(similar_campaign_data)
users_clicked_ad = len(similar_campaign_data[similar_campaign_data['AdClicked'].notnull()])

ctr = users_clicked_ad / total_users_viewed
average_session_time = similar_campaign_data['SessionTime'].mean()

print(f"Click-through rate (CTR) for similar campaigns: {ctr:.2f}")
print(f"Average session time for clicked ads from similar campaigns: {average_session_time:.2f}")

# For this example, let's plot the distribution of session times for clicked ads
plt.hist(similar_campaign_data['SessionTime'], bins=50, color='blue', alpha=0.7)

plt.ylabel('Number of Users')
plt.title('Distribution of Session Times for Clicked Ads in Similar Campaigns')
plt.grid(True)
plt.show()

# 5. Predict Performance:
# This is a simplistic prediction: we expect the new campaign to have a similar CTR and average session time.
print("Predicted Click-through rate (CTR) for the new campaign:", ctr)
print("Predicted average session time for clicked ads from the new campaign:", average_session_time)

Comments limitations & further suggestions¶

First, let's understand the calculation of Click-through rate (CTR):

$\text{CTR} = \frac{\text{Number of Ad Clicks}}{\text{Number of Total Users who viewed the ad}} $

From the code:

  • We merged the UserInteractions table with the AdInformation table based on AdClicked. This means that we only captured interactions where an ad was clicked.
  • Then, when we filtered for similar campaigns, we based it on the keywords which come from ads that were already clicked.

Therefore, the number of "Total Users who viewed the ad" and the "Number of Ad Clicks" are essentially the same for the filtered data, resulting in a CTR of 1.

"The Click-through rate (CTR) of 1 is observed because our analysis primarily focuses on interactions where ads were clicked. Given the method we used to join the tables and filter data, we inadvertently considered only successful interactions where ads were clicked. In a real-world scenario, not every ad viewed is clicked, so having a CTR of 1 is uncommon. A more robust analysis would take into account all users who viewed the ad, regardless of whether they clicked it or not."
