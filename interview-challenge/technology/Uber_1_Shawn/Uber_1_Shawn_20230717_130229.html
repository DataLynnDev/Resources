import sqlite3
import pandas as pd

# Connect to the SQLite in-memory database
conn = sqlite3.connect(':memory:')

# Convert your Pandas dataframes to SQL tables
trips.to_sql('trips', conn, if_exists='replace', index=False)
cities.to_sql('cities', conn, if_exists='replace', index=False)

# Define the queries
query1 = """
    SELECT
        cities.city_name,
        AVG(trips.actual_eta - trips.predicted_eta) as mean_diff,
        (SUM((trips.actual_eta - trips.predicted_eta -
             (SELECT AVG(trips.actual_eta - trips.predicted_eta)
              FROM trips
              WHERE
                  trips.status = 'completed'
                  AND cities.city_name IN ('Qarth', 'Meereen')
              )
             ) * (trips.actual_eta - trips.predicted_eta -
                  (SELECT AVG(trips.actual_eta - trips.predicted_eta)
                   FROM trips
                   WHERE
                       trips.status = 'completed'
                       AND cities.city_name IN ('Qarth', 'Meereen')
                   )
                  )
             )
        / COUNT(trips.id)
        ) as stddev_diff
    FROM
        trips
    JOIN
        cities ON trips.city_id = cities.city_id
    WHERE
        trips.status = 'completed'
        AND cities.city_name IN ('Qarth', 'Meereen')
    GROUP BY
        cities.city_name;
"""


# Execute the queries and store the results in Pandas dataframes
result1 = pd.read_sql_query(query1, conn)

query2 = """
    SELECT
        cities.city_name,
        strftime('%w', trips.request_at) as day_of_week,
        AVG(CASE WHEN trips.client_rating > 4 THEN 1 ELSE 0 END) as percentage_above_4
    FROM
        trips
    JOIN
        cities ON trips.city_id = cities.city_id
    WHERE
        trips.status = 'completed'
        AND trips.request_at >= '2016-01-01'
        AND trips.request_at < '2016-04-01'
        AND cities.city_name IN ('Qarth', 'Meereen')
    GROUP BY
        cities.city_name,
        day_of_week;
"""

Part 2.Experiment Design¶

Suppose the Driver Experience team is considering a new "Streak Bonus" feature, where drivers are rewarded for completing a certain number of trips in a row without cancellations.

Q3 Propose a measurable definition of 'engagement' in this context. What kind of data would you need to track this metric effectively?

Answers: In this context, 'engagement' could be defined as the number of trips a driver completes without cancellations over a given time period. This could be a week or a month, depending on the granularity of the data available. A higher number of completed trips would indicate a higher level of engagement.

Q4 Assuming we have a baseline measure of driver engagement before implementing the "Streak Bonus," how would you design an experiment to test whether this feature increases driver engagement? What statistical methods would you consider to determine if the change in engagement is significant?

Answers: To track this effectively, we would need data on each trip a driver takes, the outcome of the trip (completed or cancelled), the time the trip occurred, and whether the driver was enrolled in the "Streak Bonus" program at the time.

For the experiment design, we could use an A/B testing approach. We would randomly split drivers into two groups: a control group that doesn't have access to the "Streak Bonus" feature, and a test group that does. Both groups would be similar in terms of demographics, average completed trips per week, and other relevant metrics to reduce bias. We would then monitor both groups for a certain period (for example, a month), and measure the difference in engagement between the two groups.

For statistical analysis, we could use a two-sample t-test to compare the means of the two groups. The null hypothesis would be that there's no difference in engagement between the two groups, and the alternative hypothesis would be that there is a difference.

df = drivers_df.copy()
s_date = pd.to_datetime('2010-01-01')
# Convert dates to numerical format by extracting date features
for column in ["signup_timestamp", "bgc_date", "vehicle_added_date", "first_trip_date"]:
    df[column] = pd.to_datetime(df[column])
    df[column + 'elapsed'] = (df[column] - s_date).dt.days

# Now you can drop the original datetime columns
df = df.drop(["signup_timestamp", "bgc_date", "vehicle_added_date", "first_trip_date"], axis=1)

# One-hot encode categorical variables
encoder = OneHotEncoder()
encoded_df = pd.DataFrame(encoder.fit_transform(df[["signup_os", "vehicle_make", "vehicle_model", "city_id"]]).toarray())

# Combine the encoded features with the rest of the dataset
df = df.drop(["signup_os", "vehicle_make", "vehicle_model", "id", "city_id"], axis=1)

# Split the data into a training set and a testing set

X_test = X_test.to_numpy()
